# Infrastructure as Code для Kubernetes кластера в Yandex Cloud

Данный репозиторий содержит Terraform конфигурацию для развертывания управляемого Kubernetes кластера (Managed Service for Kubernetes) в облаке Yandex Cloud с использованием GPU-узлов.

## 🚀 Описание инфраструктуры

Инфраструктура включает в себя следующие компоненты:
- **VPC Сеть**: Настройка виртуальной сети и подсетей для кластера.
- **Kubernetes Cluster**: Основной управляемый кластер (версия 1.32).
- **GPU Node Group**: Группа прерывистых (spot) узлов с NVIDIA Tesla V100, поддерживающая автоматическое масштабирование (от 0 до 3 узлов).
- **IAM**: Создание сервисных аккаунтов и назначение необходимых ролей для корректной работы кластера.
- **ALB**: Настройка Application Load Balancer для внешнего доступа.

## 🛠 Стек технологий
- **Terraform**
- **Yandex Cloud** (Provider: `yandex`)
- **Kubernetes** (v1.32)
- **S3 Backend** (для хранения состояния tfstate)

## ⚙️ Настройка и запуск

### Переменные
Основные переменные определены в `variables.tf`:
- `cloud`: ID вашего облака в Yandex Cloud.
- `folder`: ID каталога, в котором будет развернута инфраструктура.
- `region`: Регион развертывания (по умолчанию `ru-central1-a`).

### Инструкция по развертыванию

1. **Инициализация**:
   Сконфигурируйте бэкенд в `backend.tf` (S3 bucket) и выполните:
   ```bash
   terraform init
   ```

2. **Проверка плана**:
   ```bash
   terraform plan
   ```

3. **Применение конфигурации**:
   ```bash
   terraform apply
   ```

## 📂 Структура файлов
- `main.tf` — Основные ресурсы (K8s кластер, GPU группа, SA).
- `vpc.tf` — Конфигурация сети.
- `variables.tf` — Объявление входных переменных.
- `outputs.tf` — Выходные данные после применения.
- `provider.tf` — Настройка провайдера Yandex Cloud.
- `backend.tf` — Настройка удаленного хранения состояния.
- `roles.tf` — Назначение прав доступа.
- `alb.tf` — Настройка балансировщика нагрузки.

## ⚠️ Важные замечания
- **Spot-узлы**: GPU-узлы настроены как `preemptible = true`, что значительно снижает стоимость, но означает, что узлы могут быть удалены облаком в любой момент.
- **S3 Backend**: Убедитесь, что данные в `backend.tf` заполнены корректно перед запуском `init`.
